# 痛覚評価実験

マウスの侵害受容性、痛覚応答を評価するための行動実験タスク

**バージョン**: v1.0  
**作成日**: 2025/10/18

## 実験目的
1. 侵害受容器刺激の刺激強度と痛覚応答の関係を明らかにする
2. 皮質下行性投射による侵害受容修飾が恐怖学習キネティクスに与える影響を明らかにする


## 全体方針
マウスを円柱トレッドミル上で頭部固定し、自由に歩行・走行を許す条件下で侵害刺激に対する応答を計測する。走行可能なことで拘束と侵害刺激に対するストレスの緩和が期待される。しかしながら、走行時と静止時のマウス脳の活動状態は異なるため、再現性の高い測定のためには、マウスが特定の状態（静止時か走行時）にあるときに選択的に刺激を与え応答を記録する必要がある。侵害刺激応答は静止時の方が大きく明瞭であり（予備実験データあり）、また走行時には測定不可能な応答（例：驚愕反射による屈曲運動）もある。従って、刺激は静止時に与える方針とする。静止時の刺激実験の効率を上げるためには、マウスがトレッドミル上で静止するような動機づけが必要なので（さもなくば走行し続ける個体もいる）、刺激実験に先立って、給水制限と水報酬によるトレーニングを行う。

また、本実験では、侵害刺激応答の測定の発展として、恐怖学習の学習キネティクスの評価も目指す。侵害刺激は恐怖学習の教師信号（無条件刺激、US）として機能する。大脳皮質（後部島皮質）は下行性投射を介して侵害受容性を修飾する可能性があるが、これは教師信号の修飾でもあるので、恐怖学習の進行速度や最終到達点を変化させる可能性がある。大脳皮質下行性投射の選択的操作により恐怖学習キネティクスがどのように変化するかを問う。




## トレーニングフェーズ

| フェーズ | 期間 | 目的 | 主要パラメータ |
|---------|------|------|---------------|
| **馴化** | 2–3日 | 頭固定・トレッドミル・音環境の慣れ | 給水は非随伴 |
| **P1〈lick形成〉** | 0–1日 | スパウト位置の学習 | 強い水期待を作らない |
| **P2〈Gateのみ〉** | 1日 | 静止ゲートの確立 | 成功率≥0.6、連続静止≥1.5s×20回 |
| **P0〈Baseline〉** | 1日 | 痛覚マッピング（CSなし・報酬なし） | 強度I0〜I4、強刺激≤10/日 |
| **P3〈CS−のみ〉** | 1–2日 | CS−学習（中断＋ペナルティ） | 完走率≥0.6、no-lick違反≤0.2 |
| **P4〈条件付け〉** | 1日〜 | CS+→US、CS−固定長 | US省略プローブ5%、CS長統一 |
| **P0〈Post〉** | +1日 | 再測定（学習/期待/感作の影響評価） | Day 0と同じ手順 |


## トレーニングスケジュール
**馴化 → P1 → P2 → P0（Baseline）→ P3 → P4 → P0（Post）**

> **重要**: P0はP3より前に置き、期待・連合の影響を最小化。P2で静止を確立してから臨む。



## 各フェーズの詳細説明
**馴化**
マウスに実験環境に慣れさせることを目的とする。特に、トレッドミル上での姿勢維持を覚えさせ、過剰な緊張状態の解消を目指す。給水制限や水報酬は行わない。具体的には、マウス頭部をトレッドミル上に固定し、実験環境中に３０分/日放置する。異常な姿勢を取り続ける個体は、適宜姿勢を修正してやる。マウスが自然な姿勢をトレッドミルで維持できるようになれば終了。２ー３日かかる想定。

**P1〈lick形成〉** 
マウスにトレッドミル上で水報酬が与えられることを学習させるのが目的。具体的には、マウスを２日間異常２％クエン酸水(CA)で飼育することで渇水状態にした上で、頭部固定条件でスパウトから水（純水）を得る（lick）ことを学習させる。具体的には、スパウトを口元1-2 mmに置き、2 uLの水を3秒間隔で出し続ける（state "P1_Autowater"）。自発lickが３Hz以上で起こるようになったら、報酬をlick１回につき１滴（5 uL）与えるようにする（state "P1_FR1"）。10分間で200 lick以上出れば終了。最長で30分/日。

**P2〈Gateのみ〉**
トレッドミル上で静止したときに水報酬を与えることで、静止することを動機づける。具体的には、渇水状態のマウスが静止状態を一定時間（1.0->1.5秒と段階的に伸ばす）維持し、かつlickなし状態を0.4秒維持できれば水報酬を与える（1滴 = 5 uL）。静止時に感覚フィードバックとして青色LEDを点灯する。最低20回連続で静止→報酬を達成したら終了。

**P3〈Baseline〉**
侵害刺激を与え、応答を記録する。渇水状態のマウスに対して行うが、試行中に水報酬は与えない（純粋な痛覚応答を記録するため）。静止状態を一定時間（1.5秒）維持し、かつlickなし状態を0.4秒維持できたとき、刺激を与える。刺激は0.3秒のオフセット後に与えられる。刺激強度は５段階（刺激なし、最弱、弱、強、最強）を試み、強い刺激ほど与えられる確率が低い。一定数の刺激応答記録が出来たら終了。途中でも30分で打ち切り、その後P2タスクを行って、静止の動機づけを再強化する。

**P4〈CS−のみ〉** 
静止時に水報酬と連合する条件刺激（CS-）を学習させる。渇水状態のマウスに対して行う。静止状態を一定時間（1.5秒）維持し、かつlickなし状態を0.4秒維持できたとき、CS-（純音刺激）を2.0秒間提示し、その間静止状態を維持できれば水報酬を与える。水報酬を与える直前の0.2秒間にlickがないことも報酬条件とし、後にCS-中の神経活動を記録する際、行動応答成分を抑えることを目指す。１０回連続成功（報酬獲得）を２回達成したら学習成立とみなす。最長で30分/日。


**P5〈条件付け〉** 
静止時に侵害刺激と連合する条件刺激（CS＋）を学習させる（恐怖条件付け）。渇水状態のマウスに対して行う。静止状態を一定時間（1.5秒）維持し、かつlickなし状態を0.4秒維持できたとき、CS+またはCS-（周波数の異なる純音刺激）を2.0秒間提示する。CS+の終了と同時に、75%の確率で侵害刺激が与えられる。残りの25%では刺激を与えず、条件刺激が誘起した応答（条件応答）を測定するためのプローブとする。侵害刺激の強度は比較的弱いものを採用し、条件反応の形成キネティクスが評価できるようにする。CS-はP3と同様の条件で水報酬が連合するので、マウスの文脈恐怖形成（トレッドミル上で頭部固定されている状態に対する持続的な恐怖・ストレス反応）は抑制されると期待。CS+は25%, CS-は75%のケースで提示されるようにする。


### 🐭 個体差対応

#### Runner（活動的な個体）
- P2で `v_thresh` を個体内下位10–15%分位に自動設定
- `gate_hold` 段階強化
- `gate_timeout_penalty=+2–4 s`
- 必要なら**短P3**を1回入れてから翌日にP0

#### Sitter（静止傾向の個体）
- P2の基準を緩め成功体験を増やす
- P3短縮

### 📊 進行・中断基準

| 移行条件 | 基準 |
|---------|------|
| **P2→P0** | 静止成功率≥0.6、連続静止≥1.5s×20回、Gate timeout ≤5/100試行 |
| **P0完遂** | 各強度の有効静止試行数到達、強刺激の連続/過密なし |
| **P3→P4** | CS−完走率≥0.6、ログ上ペナルティ適用を確認 |
| **倫理中断** | 強反応連続（例3回）や痛表情持続で即停止 |

## 🔬 P0：痛覚マッピング（P_noci_map）

### 🎯 基本設計
- **CS/報酬なし**で純粋な痛覚応答を測定
- **フロー**: `ITI → Gate → PreUS → IntensitySelect → US_I* → ITI`

### 📊 強度設定

| 強度ラベル | 確率 | 説明 |
|-----------|------|------|
| **I0** | 0.30 | シャム（偽陽性・期待評価用） |
| **I1** | 0.25 | 低強度 |
| **I12** | 0.20 | 中低強度 |
| **I3** | 0.15 | 高強度 |
| **I4** | 0.10 | 最高強度 |

### ⚠️ 強刺激の希少化
- **強刺激ラベル**: `["I3","I4"]`
- **最小間隔**: `strong_min_gap_trials≥5`
- **最大試行数**: `max_strong_trials_per_session≤12`

### 🛑 PreUSの厳密停止
- **保持時間**: `pre_us_hold_sec=0.3`
- **no-lick期間**: `pre_us_no_lick_sec=0.3`
- **失敗時**: 中断→ITI（ペナルティは実装側で設定）

### 🔗 同期システム
- **USオン/オフ**、**Gate/PreUS/ITI**のTTL（`event_codes`）で全系同期

## 🎵 CS/USデザイン（P3/P4の要点）

### 📉 CS−（条件刺激マイナス）
- **中断条件**: 提示中に動いたら即中断→ITI
- **報酬**: なし（失敗を明確化して学習促進）

### 📈 CS+（条件刺激プラス）
- **US提示**: CS終端で提示（共終端）または固定ISI（1.5s）で統一
- **CS長**: CS−と一致させる（1.5s または 2.0s）

### 🔍 US省略プローブ
- **分岐**: CSplusから **US (p=0.95)** / **ITI (p=0.05)** の明示分岐
- **目的**: CRの純粋評価を確保

### ⏱️ ITI（試行間隔）
- **基本時間**: U(3–6)s
- **ペナルティ**: abort/timeoutペナルティ（例: +3s / +2s）

## 💧 スパウト設計（要点）

### 🔧 物理仕様
- **先端**: 20G鈍針（外径~0.9mm）
- **位置**: 先端–舌 1–2mm、角度 20–30°
- **表面処理**: 先端3–5mmは鏡面、根元は黒化

### 📍 位置決め
- **固定**: 3軸マニピュレータ固定
- **清掃**: クイックリリースで清掃容易

### 💨 流体系
- **供給**: 一定圧供給
- **制御**: 逆止弁＋微小ベント（極細キャピラリ）でアフタードリップ抑制
- **口径**: 先端直前は ID 0.3–0.5mm

## 🖥️ pyControlへの取り込み

### 🔌 推奨接続方式
- **ハードTTL化**: 比較器＋ヒステリシス
- **入力**: Breakout 1.2 の BNC-1/2（5V対応）

### 🔄 代替接続方式
- **Analog input**: pyControlで電圧を読み取り
- **ソフト処理**: 閾値＋不応期でイベント化
- **電圧範囲**: 0–3.3Vにスケール
- **注意**: GND共有に注意

### 💻 イベント化例（デジタル入力）

```python
from pycontrol.hardware import *

board = Breakout_1_2()
lick_in = Digital_input(board.DIN_1, rising_event='Lick', debounce=5)  # 5msデバウンス
```

## ⚙️ 重要パラメータ指針

### 🏃 速度制御
- **速度閾値**: `v_thresh` は個体内速度分布の下位10–15%分位（auto-thresholds）

### 🛑 中断判定
- **スムージング**: `smoothing_window_ms=100`
- **中断条件**: `break >100ms` 連続で中断

### ⚠️ ペナルティ設定
- **CS−中断**: `cs_minus_abort_penalty_sec=3.0`
- **Gate timeout**: `gate_timeout_penalty_sec=2.0`

### 🎲 プローブ確率
- **US提示**: `p_us_on_cs_plus=0.95`
- **US省略**: `p_us_omit_on_cs_plus=0.05`
- **管理**: YAMLのglobalsで一元管理

## ⚠️ 付録：よくある落とし穴と対処

### 🎵 CS/US関連
- **CS+/-の長さ不一致** → **同一長に統一**（1.5s or 2.0s）

### 🔊 音響問題
- **ソレノイド音の混入** → バルブを装置外＋防振、最小パルス幅＆不応期で誤検出回避

### 💧 水システム
- **アフタードリップ** → **逆止弁＋微小ベント**＋細径下流チューブ、一定圧供給

### ⚡ 電気ノイズ
- **グラウンドループ** → センサ系は電池駆動、**デジタル絶縁**でpyControlへ
